# 福生仓线机制说明：入口、潜入、逃出、医院

> 用途：这份文档专门说明《雾锁迷踪》中福生仓线从开始到医院承接的机制，便于人工核查。
>
> 核查范围：
>
> - 福生仓线如何开启
> - 直接行动、老孙支援、SOLO 独自潜入三种模式如何分流
> - 仓库潜入如何按时间压力、heat、支援状态分流
> - 暗门、救人、逃出如何判定
> - 医院线如何承接码头结果，并影响结局质量

---

## 1. 福生仓线如何开始

福生仓线主要从 `线索整理 / ch3_wrapup` 开始开放。

### 1.1 基础入口

玩家在 `ch3_wrapup` 中，如果满足以下条件：

```text
未找到沈玉芳
且未错过 deadline
且已经掌握王巡官纸条或福生仓位置线索
```

则应出现：

```text
⛵ 去苏州河废弃码头——查福生仓
```

核心判断可以理解为：

```text
!found_yufang
&& !missed_deadline
&& (got_wang_note || hasClue('福生仓位置'))
```

### 1.2 福生仓入口线索来源

福生仓入口不应只有一条来源，至少应支持以下来源：

| 来源 | 关键线索/道具 | 作用 |
|---|---|---|
| 巡捕房王巡官线 | `王巡官遗留纸条`、`半张烟盒纸` | 指向“福生仓，三日清” |
| 薛华立路看门人线 | `福生仓标识`、`福生仓地址`、`福生仓位置` | 地图或看门人补充出仓库位置 |
| 码头现场线 | `福生仓位置` | 已知仓库地点后可直接行动 |

核查时要确认：

```text
只拿王巡官纸条，能否去福生仓？
只拿福生仓标识/地址，能否去福生仓？
两个都拿到时，入口是否稳定出现？
```

---

## 2. 福生仓行动的三种模式

福生仓不是单一路线，进入前后应分为三种行动模式。

| 模式 | 入口选择 | 特点 | 风险 |
|---|---|---|---|
| SOLO 独自潜入 | 不找支援，独自从东侧窗户潜入 | 快，无外部支援，后续潜入/撤离/医院都要自己控场 | 暴露、撤离、医院压力更高 |
| 老孙快速支援 | 说服老孙，只带一个便衣立刻去 | 比 SOLO 稳一点，仍然较快 | 人手少，压不住傅启元 |
| 老孙完整支援 | 调齐人手再行动 | 控场最强，可封锁码头 | 耗时，可能错过完整搜查窗口 |

## 3. 老孙支援机制

### 3.1 找老孙商量福生仓

当玩家拥有以下任一内容时，应可在 `ch3_wrapup` 看到：

```text
🚓 去巡捕房找老孙商量福生仓
```

可用条件包括：

```text
福生仓地址
半张烟盒纸
王巡官遗留纸条
福生仓位置
```

进入 `ch4_sun_support` 后，老孙不会无条件支援。玩家需要出示证据。

### 3.2 出示证据给老孙

可出示：

```text
福生仓地址
半张烟盒纸
```

成功后进入：

```text
ch4_sun_present_fusheng
```

应获得：

```text
老孙愿意私下支援
sun_support_available = true
```

### 3.3 快速支援

选择：

```text
🏃 只带一个便衣立刻赶去福生仓（快，人少）
```

应设置：

```text
sun_fast_support = true
sun_support_in_action = true
```

机制含义：

- 速度快。
- 有便衣帮忙，但不是完整封锁。
- 撤离时可以拖住傅启元一会儿。
- 医院控制力只小幅提高。

### 3.4 完整支援

选择：

```text
🚓 调齐人手再行动（稳，但慢，也更易惊动）
```

应设置：

```text
sun_full_support = true
sun_wait_support = true
sun_support_in_action = true
```

机制含义：

- 控场强。
- 可封锁码头。
- 医院控制力高。
- 但消耗时间，可能导致福生仓部分清场或完全清场。

核查重点：

```text
快速支援不应等同完整支援。
完整支援应明显提高撤离/医院控制，但应有时间代价。
```

## 4. SOLO 独自潜入机制

SOLO 线不是“无支援数值”，而是一条独立流程。

### 4.1 SOLO 入口

入口文案应明确：

```text
🔦 不找支援，独自从东侧窗户潜入
```

点击后应设置：

```text
dock_force_solo_entry = true
dock_solo_entry_requested = true
```

同时必须清除所有支援 flag，避免误判成老孙线：

```text
sun_fast_support
sun_fast_support_active
sun_fast_cover_escape
dock_fast_support_entry
sun_full_support
sun_wait_support
sun_support_in_action
dock_full_support_entry
dock_full_support_tradeoff
dock_blockade_record
```

### 4.2 SOLO 强制路由

`routeDockByPressure()` 应优先识别 SOLO：

```text
如果 dock_force_solo_entry 为 true
且尚未正式进入仓库
→ 返回 ch4_dock_solo_infiltration
```

核查重点：

```text
选择 SOLO 后，不应直接进入 ch4_dock_full_search。
选择 SOLO 后，不应出现老孙/便衣支援文本。
```

---

## 5. 潜入分流机制

福生仓潜入不是固定节点，而是根据以下因素分流：

```text
时间压力 deadlinePhase
+ heat / 暴露惩罚
+ 支援缓解
+ SOLO 额外暴露/拖延
```

### 5.1 基础时间阶段

| 阶段 | 含义 | 入口结果 |
|---|---|---|
| safe | 时间较充裕 | 完整搜查 |
| tight | 时间吃紧 | 有限搜查 |
| critical | 只够救人 | 只够救人 |
| expired | 已迟到 | 仓库清场 |

### 5.2 普通仓库入口分流

`routeDockByPressure()` 应大致分流为：

```text
safe     → ch4_dock_full_search
tight    → ch4_dock_limited_search
critical → ch4_dock_rescue_only
expired  → ch4_dock_cleared
```

### 5.3 暗门深入分流

`routeDockDeepByPressure()` 应大致分流为：

```text
safe     → ch4_dock_deep_dual
tight    → ch4_dock_deep_trace
critical → ch4_dock_deep_rescue_only
expired  → ch4_dock_cleared
```

### 5.4 heat 与支援修正

机制应体现：

```text
heat 越高，搜索窗口越差。
支援越强，能抵消部分风险。
完整支援 > 快速支援 > SOLO。
```

核查重点：

```text
同样进入福生仓，早到/晚到结果是否不同？
同样时间，SOLO/快速支援/完整支援结果是否不同？
高 heat 是否把 safe 推成 tight 或 critical？
```

## 6. SOLO 东窗巡灯机制

SOLO 不应直接进入普通仓库搜索，而应先进入：

```text
ch4_dock_solo_infiltration
→ ch4_dock_solo_window_crisis
```

### 6.1 ch4_dock_solo_infiltration

应有选项：

```text
🔦 贴近东窗，先看清守卫巡灯
```

点击后进入：

```text
ch4_dock_solo_window_crisis
```

### 6.2 东窗巡灯三选项

| 选项 | flag | 机制含义 |
|---|---|---|
| `🤫 压低身子，等巡灯从窗框上移开` | `dock_solo_waited_patrol` | 暴露低，但拖延增加 |
| `🧺 扯一块油布挡住窗影，马上翻进去` | `dock_solo_window_screen` | 小幅增加暴露，节省时间 |
| `⚠️ 不停下来，趁装车声直接翻进仓库` | `dock_solo_forced_window` | 暴露明显增加，但最快 |

三个选项都应进入：

```text
routeDockSearchByTime()
```

或等价的仓库搜索分流。

核查重点：

```text
routeDockSearchByTime 是否存在？
三个选项是否都能继续？
dock_solo_waited_patrol 是否增加 delay？
dock_solo_window_screen 是否增加 exposure？
dock_solo_forced_window 是否增加更多 exposure？
```

## 7. 仓库搜索机制

### 7.1 完整搜查：safe

节点应类似：

```text
ch4_dock_full_search
```

应获得：

```text
福生仓位置
公董局公文纸
清场指令
```

后续应可选择：

```text
查教具箱
直接找声音来源 / 暗门
```

### 7.2 有限搜查：tight

节点应类似：

```text
ch4_dock_limited_search
```

应获得：

```text
福生仓位置
公董局公文纸
清场指令
```

但时间更紧，证据和救人窗口应受到限制。

### 7.3 只够救人：critical

节点应类似：

```text
ch4_dock_rescue_only
```

应获得：

```text
福生仓位置
福生仓仓促清场
```

应直接导向救人/暗室，不应还有完整搜查空间。

### 7.4 迟到清场：expired

节点应类似：

```text
ch4_dock_cleared
```

应获得：

```text
福生仓清场
苏晚亭半张字条
```

并设置：

```text
missed_deadline = true
```

## 8. 教具箱与暗门机制

### 8.1 教具箱

节点：

```text
ch4_dock_crates
```

应获得：

```text
教具箱走私
管制药品走私
光华货运单
铁钎
found_door_tool = true
```

### 8.2 暗门

节点：

```text
ch4_dock_locked_door
```

根据是否有 `铁钎` 分两种：

| 条件 | 选项 | 后果 |
|---|---|---|
| 有 `铁钎` | `🧰 用铁钎撬开暗门` | 正常进入暗室分流 |
| 无 `铁钎` | `⚠️ 砸锁开门` | `addHeat(2)` 后进入暗室分流 |
| 无 `铁钎` | `📦 回头检查教具箱找工具` | 回到教具箱拿工具 |

核查重点：

```text
先查教具箱，暗门是否变成用铁钎？
不查教具箱，是否只能砸锁或回头找工具？
砸锁是否加 heat？
```

---

## 9. 暗室结果机制

暗室结果由 `routeDockDeepByPressure()` 决定。

| 节点 | 条件倾向 | 结果 |
|---|---|---|
| `ch4_dock_deep_dual` | safe | 同时找到沈玉芳和苏晚亭 |
| `ch4_dock_deep_trace` | tight | 找到沈玉芳，苏晚亭刚被转走，留下学生证/字条 |
| `ch4_dock_deep_rescue_only` | critical | 找到沈玉芳或痕迹，苏晚亭只剩手表等证据 |
| `ch4_dock_cleared` | expired | 仓库已清空 |

### 9.1 双救结果

应设置：

```text
found_yufang = true
found_su_at_dock = true
```

后续逃出完成后应设置：

```text
rescued_yufang = true
rescued_su = true
```

### 9.2 单救/痕迹结果

可能设置：

```text
found_yufang = true
su_moved_from_dock = true
```

或：

```text
su_trace_only = true
```

核查重点：

```text
双救、单救、只拿痕迹是否真的有不同 flag？
后续医院是否根据证人人数变化？
```

## 10. 逃出机制

逃出阶段应根据行动模式分流。

---

### 10.1 支援线逃出

如果有老孙/便衣支援，逃出时可出现：

```text
让老孙的人亮明身份
让老孙的人封住码头
让便衣拖住傅启元
借雾绕开汽车
当场质问傅启元
```

系统应根据以下条件过滤或添加选项：

```text
是否有支援
支援是快速还是完整
是否有硬证据
heat 是否过高
```

### 10.2 完整支援的额外收益

完整支援或码头封控成功时，应获得：

```text
码头封锁记录
码头看守口供
```

这类证据应提高后续医院控制与结局质量。

### 10.3 快速支援的特点

快速支援通常只能：

```text
便衣拖住傅启元
掩护撤离
```

不应直接等同于完整封锁码头。

---

## 11. SOLO 逃出机制

SOLO 下撤离选项应替换为 SOLO 专属四选项。

| 选项 | flag/效果 | 后果 |
|---|---|---|
| `🌫️ 沿水边栈道绕开黑车，先把人带走` | `dock_solo_waterline_escape`，线索：水边栈道撤离 | 低调撤离，但医院压力略增 |
| `🧱 推倒空木箱挡住车灯，争出几步空隙` | `dock_solo_crate_screen`，`addHeat(1)`，线索：木箱遮灯撤离 | 动静更大，撤离紧张度上升 |
| `⚠️ 让证人先藏进货车阴影，自己引开视线` | `dock_solo_decoy_escape`，`addHeat(1)`，线索：孤身引开视线 | 后续医院压力更重，证人稳定下降 |
| `⚠️ 站到车灯前，当场拿出货运单和清场指令` | `dock_solo_hard_confront`、`dock_fast_confront_hard_evidence`、`dock_confront_fu`，`addHeat(4)` | 可能触发 `end_dock_silenced`，否则进入傅启元对峙 |

核查重点：

```text
SOLO 撤离时不应出现“老孙的人亮明身份”。
SOLO 硬对峙是否可能进入 end_dock_silenced？
水边撤离、木箱遮灯、引开视线是否影响医院评分？
```

## 12. 逃出完成后的医院入口

逃出成功后通常进入：

```text
ch4_dock_escape_finish
```

该节点应根据是否找到苏晚亭显示不同文本：

| 条件 | 结果 |
|---|---|
| `found_su_at_dock = true` | 文本体现沈玉芳和苏晚亭一起被带离 |
| 无 `found_su_at_dock` | 文本体现只带沈玉芳离开，苏晚亭仍在他们手里或刚被转走 |

逃出完成应设置：

```text
rescued_yufang = true
```

如果已找到苏晚亭，还应设置：

```text
rescued_su = true
```

然后进入医院/线索整理/后续结案流程。

核查重点：

```text
双救与单救文本是否不同？
rescued_yufang 是否设置？
rescued_su 是否只在真的救出苏晚亭时设置？
```

## 13. 医院机制总览

医院不是固定剧情，而是根据码头结果生成状态。

医院机制分四个层面：

```text
证人情况 hospitalWitnessProfile
医院压力 hospitalPressureScore
医院控制 hospitalControlScore
证人稳定 witnessStabilityScore
```

再组合出：

```text
医院状态 hospitalOutcomeTier
真相完整度 truthCompletenessTier
```

---

## 14. 证人情况 hospitalWitnessProfile

证人情况由是否救出沈玉芳、苏晚亭决定。

| 状态 | 条件 | 含义 |
|---|---|---|
| 双证人 | 沈玉芳 + 苏晚亭都在 | 证词最完整，但医院压力也更高 |
| 单证人 | 只有沈玉芳 | 医院更易控，但缺苏晚亭本人证词 |
| 证人缺席 | 两人都没救出 | 真相完整度很低 |

注意：

```text
双救 + 紧张医院：合理。
单救 + 可控医院：合理。
医院状态和真相完整度不是同一件事。
```

---

## 15. 医院压力 hospitalPressureScore

医院压力主要受以下因素影响：

| 因素 | 影响 |
|---|---|
| 码头危机值高 | 压力增加 |
| 双证人 | 压力高于单证人 |
| 正面对峙傅启元 | 压力增加 |
| 码头撤离混乱 | 压力增加 |
| 过早让陆念薇现身 | 压力增加 |
| 逼问沈玉芳 | 压力增加 |
| 强迫苏晚亭立刻指认 | 压力大幅增加 |
| 分开保护证人 | 压力降低 |
| 医生伤情记录 | 压力降低 |

---

## 16. 医院控制 hospitalControlScore

医院控制主要受以下因素影响：

| 因素 | 影响 |
|---|---|
| 完整支援 | 控制大幅提高 |
| 快速支援/便衣掩护 | 控制小幅提高 |
| 分开保护证人 | 控制提高 |
| 医生伤情记录 | 控制提高 |
| 码头封锁 | 控制提高 |
| 陆念薇交给老孙 | 控制提高 |
| 同时拥有清场指令和光华货运单 | 控制提高 |

---

## 17. 证人稳定 witnessStabilityScore

证人稳定主要受以下因素影响：

| 因素 | 影响 |
|---|---|
| 救出沈玉芳 | 稳定增加 |
| 救出苏晚亭 | 稳定增加 |
| 码头撤离不稳 | 稳定下降 |
| 正面对峙傅启元 | 稳定下降 |
| 分开保护证人 | 稳定增加 |
| 医生伤情记录 | 稳定增加 |
| 逼问沈玉芳 | 稳定下降 |
| 过早让陆念薇现身 | 稳定下降 |
| 强迫苏晚亭立刻指认 | 稳定大幅下降 |

---

## 18. 医院状态 hospitalOutcomeTier

医院状态由：

```text
hospitalCrisisScore = hospitalPressureScore - hospitalControlScore
```

与 witness 分数组合决定。

| 状态 | 机制含义 |
|---|---|
| 稳定医院线 | 压力被控制住，证人稳定 |
| 可控医院线 | 有压力，但还能控住 |
| 紧张医院线 | 证人还在，但风险明显 |
| 医院失控 | 压力过高或证人状态过差 |

核查重点：

```text
不同码头撤离方式是否导致不同医院状态？
保护证人/医生记录是否能改善医院状态？
强迫指认是否能拉低医院状态？
```

---

## 19. 真相完整度 truthCompletenessTier

真相完整度不等于医院状态。

它主要看：

| 因素 | 影响 |
|---|---|
| 救出沈玉芳 | 增加完整度 |
| 救出苏晚亭 | 增加完整度 |
| 清场指令/公董局公文纸 | 增加完整度 |
| 光华货运单/教具箱走私 | 增加完整度 |
| 医生伤情记录 | 增加完整度 |
| 陆念薇供述或交给老孙 | 增加完整度 |

重要限制：

```text
如果没有苏晚亭本人证词，真相完整度应有上限。
```

也就是说：

```text
单救路线医院可以很稳，但真相完整度不应无限高。
```

## 20. SOLO 医院机制

SOLO 到医院后，应出现特殊文本：

```text
后门没有便衣，巷口没有巡捕，病房外也没有老孙的人。
```

### 20.1 SOLO 医院四选项

| 选项 | flag/效果 | 机制含义 |
|---|---|---|
| `🛏️ 先把证人送进病房，压低声音等医生` | `hospital_triage_settle_witness`，线索：医院后门安置 | 稳妥低调 |
| `🚪 托值夜护士锁住后门，先别让外人进来` | `hospital_triage_solo_lock_backdoor`，线索：护士锁住医院后门 | 降低压力，提高控制和稳定 |
| `⚠️ 让周怀安立刻进来认人` | `hospital_triage_zhou_early`，`addHeat(1)` | 情绪冲击，增加风险 |
| `🚶 直接去走廊面对所有人的争执` | `hospital_triage_direct_corridor`、`hospital_triage_solo_no_guard` | 无守卫控场，压力上升 |

### 20.2 SOLO 对医院评分的额外影响

| SOLO 行为 | 医院影响 |
|---|---|
| 水边撤离 | 医院压力增加 |
| 木箱遮灯撤离 | 医院压力增加 |
| 自己引开视线 | 医院压力明显增加，证人稳定下降 |
| 护士锁后门 | 医院压力下降，控制和稳定上升 |
| 医院无人守门 | 医院压力上升 |

核查重点：

```text
SOLO 进医院后是否出现专属四选项？
护士锁门是否真的改善 pressure/control/witness？
周怀安过早认人是否加 heat？
直接走廊争执是否设置 hospital_triage_solo_no_guard？
```

## 21. 医院冲突后的选择

医院冲突节点应根据当前状态提供后续处理方式，例如：

```text
分开保护证人
医生做伤情记录
让老孙封锁码头 / 补封码头
逼陆念薇现身
让苏晚亭立刻指认傅启元
```

这些选择不应只是剧情文字，而应改变：

```text
hospitalPressureScore
hospitalControlScore
witnessStabilityScore
truthCompletenessTier
```

核查重点：

```text
分开保护证人是否提高稳定？
医生记录是否提高控制与真相完整度？
强迫苏晚亭指认是否明显拉低稳定？
有无老孙支援是否影响“封码头”选项？
```

## 22. 福生仓线对三道推理题的影响

福生仓线主要影响第三道推理题：

```text
推理——福生仓与公董局的真相
```

需要的证据包括：

```text
王巡官遗留纸条 / 半张烟盒纸 / 福生仓地址 / 福生仓位置
陈明远的信 / 陈明远的退缩
恐吓信
公董局公文纸 / 清场指令 / 福生仓清场
教具箱走私 / 管制药品走私 / 光华货运单 / 光华小学采购疑点
```

如果玩家只走 critical 或 expired 路线，可能缺少现场证据，从而无法触发或无法高质量完成第三道推理。

核查重点：

```text
完整搜查后是否能触发第三道推理？
有限搜查是否仍有替代证据？
只够救人是否因证据不足而影响推理？
迟到清场是否只允许低质量结案？
```

## 23. 福生仓线对结局质量的影响

结局质量不应只看是否救人，而应综合：

```text
是否救出沈玉芳
是否救出苏晚亭
是否拿到清场指令 / 公董局公文纸
是否拿到光华货运单 / 教具箱走私
是否完成第三道推理
医院状态是否稳定
证人是否稳定
真相完整度是否足够
是否有老孙封锁记录 / 看守口供
```

### 23.1 高质量结局倾向

通常需要：

```text
双救或至少证人保护充分
现场证据足够
医院线稳定或可控
第三道推理完成
学校三证物质询完成
```

### 23.2 低质量结局倾向

可能来自：

```text
错过福生仓 deadline
只拿到痕迹，没有救出关键证人
SOLO 强行对峙导致码头失控
医院失控
强迫证人导致稳定度过低
缺少现场硬证据
```

## 24. 人工核查推荐路线

### 24.1 老孙完整支援线

```text
线索整理
→ 找老孙商量福生仓
→ 出示半张烟盒纸/福生仓地址
→ 调齐人手再行动
→ 福生仓
→ 搜索仓库
→ 查教具箱
→ 开暗门
→ 救人
→ 让老孙封住码头
→ 医院
→ 分开保护证人
→ 医生伤情记录
→ 结论页
```

预期：

```text
控制力强，现场证据多，但可能因耗时影响救人/搜证窗口。
```

### 24.2 老孙快速支援线

```text
线索整理
→ 找老孙商量福生仓
→ 出示证据
→ 只带一个便衣立刻赶去
→ 潜入仓库
→ 查教具箱
→ 开暗门
→ 救人
→ 便衣拖住傅启元 / 借雾撤离
→ 医院
```

预期：

```text
速度快，风险中等，控制力弱于完整支援。
```

### 24.3 SOLO 低调线

```text
线索整理
→ 去福生仓
→ 不找支援，独自从东侧窗户潜入
→ 东窗巡灯
→ 等巡灯或油布遮影
→ 搜仓库
→ 查教具箱
→ 开暗门
→ 救人
→ 沿水边栈道绕开黑车
→ 医院
→ 托护士锁住后门
→ 分开保护证人 / 医生记录
→ 结论页
```

预期：

```text
没有支援，但如果处理稳，可以进入较高质量结局。
```

### 24.4 SOLO 高风险线

```text
线索整理
→ 去福生仓
→ 不找支援，独自从东侧窗户潜入
→ 强行翻窗
→ 查教具箱
→ 暗门
→ 救人
→ 孤身站到车灯前硬对峙傅启元
→ 如果未被灭口，则医院
→ 让周怀安立刻认人 / 直接走廊争执
→ 结论页
```

预期：

```text
可能进入 end_dock_silenced，或导致医院压力极高、结局质量下降。
```
