# 响应式 UI 与分页线索簿改造

## 背景

旧版界面采用单列布局：

- 主剧情区域固定在中间；
- 线索簿通过右侧抽屉打开；
- 线索、物品、人物、足迹、结局全部纵向堆叠。

问题是：

- 桌面端宽屏空间利用不足；
- 手机端线索簿内容过长；
- 线索、物品、人物混在同一个滚动流里，后期内容越多越难找。

---

## 本次改动

### 1. 新增响应式 UI 模块

新增：

```text
src/story-ui-responsive.js
```

它不改剧情，不改存档，只增强 UI：

- 接管 `renderPanel()`；
- 给线索簿增加分页标签；
- 桌面端让信息栏常驻右侧；
- 手机端保留底部弹层，但内容分页显示。

### 2. 接入故事模块入口

`src/story-modules.js` 新增加载：

```text
src/story-ui-responsive.js
```

UI 增强也作为正式故事模块的一部分管理。

### 3. 线索簿分页

线索簿拆成六个页面：

```text
总览
线索
物品
人物
足迹
结局
```

总览页只显示：

- 案情判断；
- 调查压力；
- 线索数量；
- 人物发现数；
- 推理墙；
- 最近线索。

大量线索、物品、人物不再全部堆在同一页。

### 4. 桌面端布局

宽屏下：

- 左侧为主剧情区；
- 右侧为常驻故事面板；
- 历史日志限制高度，避免把当前场景挤出屏幕；
- 当前场景卡片可独立滚动。

目标是尽量在一屏内同时看到：

- 当前剧情；
- 选择按钮；
- 案情总览；
- 关键线索入口。

### 5. 手机端布局

小屏下：

- 操作按钮移动到底部；
- 状态栏吸顶；
- 线索簿从底部弹出；
- 标签页横向滑动；
- 每个分页内部独立滚动。

这样手机上不会出现一个超长线索簿把内容全部挤在一起的问题。

---

## 兼容性说明

`story-ui-responsive.js` 会检测完整运行时是否存在。

如果是在 Node 测试环境里的轻量 stub，它会自动跳过，不影响现有剧情测试。

---

## 后续建议

后续如果线索继续增多，可以继续加：

- 线索搜索；
- 按人物 / 地点 / 时间分类；
- 已使用 / 未使用证据标记；
- 当前场景推荐可出示证据。
